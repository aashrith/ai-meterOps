# C4 Diagram

This diagram shows the initial shape of the service using a DDD + hexagonal split.

```mermaid
C4Context
title AI Usage Metering and Quota Service

Person(client, "API Client", "Calls the service to generate text and inspect usage")
System_Boundary(service, "AI Usage Metering Service") {
    System(api, "FastAPI Application", "HTTP API, validation, request orchestration")
    Container(ai, "AI Provider Adapter", "Mocked or pluggable LLM backend")
    ContainerDb(db, "Postgres", "User config, reservations, and usage ledger")
}

Rel(client, api, "HTTPS/JSON")
Rel(api, ai, "Generate prompt and receive usage")
Rel(api, db, "Read and write quota state")
```

## Reading the diagram

- The HTTP API is the inbound adapter.
- The application and domain layers own quota rules, credit math, and reconciliation.
- Postgres is the durable system of record.
- The AI adapter is pluggable so the backend can stay mocked during development or switch providers later.
