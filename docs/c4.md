# C4 Diagrams

These diagrams show the service from two angles: the external context and the internal container split.

## Context View

```mermaid
flowchart LR
    client([API Client])
    service["AI Usage Metering Service"]
    ai[[AI Provider]]

    client -->|HTTPS / JSON| service
    service -->|generation requests| ai
```

## Container View

```mermaid
flowchart LR
    client([API Client])

    subgraph service["AI Usage Metering Service"]
        api[FastAPI API]
        core[Metering Core]
        db[(Postgres)]
    end

    ai[[AI Provider]]

    client -->|HTTPS / JSON| api
    api --> core
    core --> db
    core --> ai
```

## Reading the diagrams

- The context view shows what talks to what.
- The container view shows how the service is split.
- FastAPI stays thin.
- The core owns quota and credit rules.
- Postgres is the source of truth.
- The AI backend is pluggable, so mock and real providers can be swapped later.
